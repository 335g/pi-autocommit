## 背景
安価モデルは Conventional Commits の type（feat/fix/chore）を誤ることが多い。
`inferTypeFromFiles()` で推論した型をヒントとしてプロンプトに注入する施策。

## レビュアー判断
✅ 実施推奨。ただし:
- システムプロンプトではなく**ユーザープロンプト**に注入
- 「参考情報。判断はモデルに委ねる」というディスクレーマー付き
- ファイルレベルのヒントは出さない（グループ化を誘導しないよう）

## 実装判断の観点
- 効果測定で `msgSanitizeChanged`（形式違反）が多ければ実装価値が高い
- 効果測定で `msgIsGeneric` が多ければ型アンカーより分解再生成を優先
- 安価モデルでの `isConventionalCommit` 通過率が低ければ型アンカーが有効

## 実装方針（実施する場合）
1. `diff-analyzer.ts` の `buildPrompt()` で、diff 内のファイル拡張子から型候補を計算
2. `{typeHints}` プレースホルダをプロンプトテンプレートに追加
3. `i18n/messages.ts` の `diffAnalyzer.buildPrompt` に `{typeHints}` を追加
4. ヒント文言: "Hint: Based on file analysis, possible types: feat, docs (use your judgment)"
