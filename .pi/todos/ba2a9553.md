## 背景
安価モデルの最大の弱点「会話意図の誤抽出」に対処する。
1回目の生成結果が汎用的だった場合、タスクを分割して再試行:
- ユーザーメッセージのみ → 意図抽出
- アシスタント最終応答のみ → 変更内容要約
- 2つの結果をテンプレート合成

## レビュアー判断
✅ 実施推奨。費用対効果が最も高い P1 施策。

## 実装判断の観点
- `msgIsGeneric` が高頻度 → 実施価値大
- `msgRefineTriggered` が高頻度 → 既存の refine 機構では不十分、分解再生成が必要
- API コスト: 最悪3回のAPIコール（1回目失敗→2回分解）。GPT-4o-mini で ~$0.001

## 実装方針（実施する場合）
1. `auto-commit-message.ts` に `generateDecomposedMessage()` 関数を追加
2. `generateAutoCommitMessage()` 内で、生成結果が `isGenericMessage()` にヒットした場合に分解パスへ
3. 分解パス:
   - `extractIntent(userMessages)`: 最後のユーザーメッセージから意図を抽出
   - `summarizeResponse(assistantMessages)`: 最後のアシスタント応答から変更内容を要約
   - `synthesizeMessage(intent, summary, files)`: テンプレート合成
4. プロンプトは既存のシステムプロンプトを流用（例が既に入っているため）
