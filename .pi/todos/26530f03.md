## 目的
P0 施策が高価モデル（Sonnet 級）の出力を劣化させていないことを確認する。

## 手順
1. `analysis_model` を解除（セッションモデル = Sonnet に戻す）
2. `/git-agg-commit` を2〜3回実行
3. `/git-diagnostics` で確認:
   - Layer 3+4 が 0 であること（リペア不要 = Sonnet は正常JSONを出力）
   - `msgSanitizeChanged` が 0 に近いこと
4. 生成されたコミットメッセージの品質を目視確認（汎用でないか）

## 判断基準
- Layer 3+4 > 0 → 何かおかしい（調査必要）
- `msgSanitizeChanged` 増加 → プロンプトが Sonnet の出力を悪化させている可能性
